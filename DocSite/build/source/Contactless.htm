SDK preview

The MaSDK is a SDK for QR code encoding and decoding in Alipay plus product .Usually used for contactless payment method as a face to face near field communication technology implementation. Comparing to sonic wave payment, QR code could be printed as a hard copy and contains more information for conplex payment scenario . MaSDK enables developer to implement QR code application and developing face to face payment ability with other component in Alipay plus open API or transfer module.

MaSDK contains an encoder agent and a decoder agent for developrt to use in their UI, it also contains a well functioned scanning Activity that could be started by user using intents and provide scanning result by Activity result .

Getting Start

MaSDK provided as masdklite-1.0.aar library containing all its code, resources, and native libraries.

 

1¡¢Integration

     To support using aar as library, one should use a repository to reference an aar library as dependency for a project. The most simple way is to define a local floder as repository in your project . And refer to this repository using aar ext.

    
repositories {
    flatDir {
        dirs 'libs' //this way we can find the .aar file in libs folder
    }
}

dependencies {
    compile fileTree(dir: 'libs', include: ['*.jar'])
    compile 'com.android.support:appcompat-v7:23.1.1'
    compile (name:'masdklite-release', ext:'aar')
}

 

2¡¢Permissions

MaSDK use mobile phone camera to get data for decoding. It also requires local external storage permission to run with full funtion. Add these permission to android manifest .



<uses-permission android:name="android.permission.CAMERA" />

<uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE" />

<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE" />

 

3¡¢Start QR code scanning by MaCaputreActivity

The MaCaptureActivity needs a vibrate permission to vibrate user¡¯s phone when scanning result got , you need to add this permission to your manifest before using this activity , other refer to the next chapter for writing your own defined Capture Activity.

 

<uses-permission android:name="android.permission.VIBRATE"/>

 

After perssion granted , You could start our predefined MaCaptureActivity by an intent and wait for scanning results.

 

Intent intent = new Intent("android.intent.action.MAIN");

intent.setClassName(this, MaCaptureActivity.class.getName());

startActivityForResult(intent, MaCaptureActivity.REQ_CAPTURE);

By process the result for scanning you can go through to payment or friend relationship operations according to QR code data in result bundle.

 

@Override

protected void onActivityResult(int requestCode, int resultCode, Intent data) {

    switch (resultCode){

        case 101:

            //this result code shows data contains a QR code string

            String code = data.getStringExtra("decodeResult");

            String perf = data.getStringExtra("perf");

            break;

        case 104:

            //this result code is a normal return from scanning

            break;

    }

}

 

You can get the code and log of this scanning by bundle extras in activity results , however you need to write your own scanning activity when special funtion or visual style is needed.

Decoding by a decoding agent

In your own activity , you code could use a decoding agent to decoding QR code captured or using other input methods.

 

1¡¢Camera preview

Firstly ,you should implement a basic camera preview , you can refer to this document to start a basic camera activity .

 http://developer.android.com/training/camera/cameradirect.html

Also an IAPScannerPreview has been provided in our library which could start camera by its own, so developer could just place this view on his layouts to build a previewed widget as a part of capture activity .

Use a RectViewFinder also for use to aimming their camera to the code in the middle region.

<com.alipay.imobile.android.masdklite.widget.IAPScannerPreview

    android:id="@+id/svScan"

    android:layout_width="match_parent"

    android:layout_height="match_parent" />



<com.alipay.imobile.android.masdklite.widget.RectViewFinder

    android:id="@+id/viewfinder"

    android:layout_width="240dp"

    android:layout_height="240dp"

    android:layout_centerInParent="true" />

 

To enable the preview widget , you should bind a camera manager to the preview for this view to open and show camera frames from the manager.

 

scannerPreview = (IAPScannerPreview) findViewById(com.alipay.imobile.android.masdklite.R.id.svScan);
scannerPreview.setCameraManager(new CameraManager(MainActivity.this
));

 

2¡¢Processing camera frames

By processing camera frames with a decoder agent user can decode QR code and get code contents.

Decoder agent is a handler accepts messages contains a frame of data , using the following format to pass a fram to decoder , decoder will process this data and call its listener when got QR code inside a frame of picture. Implements Camera.PreviewCallback and using as the following code.

 

scannerPreview.previewCallback = callbackListener;

 

@Override

public void onPreviewFrame(byte[] data, Camera camera) {

    if (decodeAgent == null) {

        try {

            decodeAgent = new DecodeAgent(camera, this);

        } catch (ClassCastException e) {

            Log.e("MainActivity", e.getMessage());

        }

    }



    Message message = decodeAgent.obtainMessage();



    Bundle dataBundle = new Bundle();

    dataBundle.putByteArray(KeyConstant.KEY_DATA, data);

    dataBundle.putInt(KeyConstant.KEY_HEIGHT, rectViewFinder.getHeight());

    dataBundle.putInt(KeyConstant.KEY_WIDTH, rectViewFinder.getWidth());

    dataBundle.putInt(KeyConstant.KEY_FORMAT, KeyConstant.MaIntType.MA_QR);



    message.setData(dataBundle);

    message.sendToTarget();

}

 

The height and width of target QR code zone are needed , and suggested to be identical to the rectViewFinder, so that if user adjusting the preview screen to match the rect , Decoder will likely to get a decoding result.

 

The format of a targt QR code is also needed , for the decoder to guess the target is a QR code or a bar code ,the decoder will process frame data in different way according to format defined inside the bundle.

 

Also , data should be contains in the bundle for decodding.

 

3¡¢Receiving decode result

Once a frame sent to decoder agent , the decoder will decode and try identify QR code in a QR code bitmap , if success , this decoder will trigger a listener and return with decoded a String.

To register the ScanResultCallback before scanning a QR code ,which is usuall in the following format

 

public void onScanFinish(boolean isSuccess, String result)

 

If the success flag is true , you can use the result for going to the next step, either finish the activity with result code , or do a jump to other Activity.
